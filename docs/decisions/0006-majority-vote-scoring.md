# ADR-0006: Majority vote для стабильности оценок

**Status:** Accepted  
**Date:** 2026-04-08

## Context

GLM-4.7 (Z.AI) при temperature=0 всё ещё показывает значительный разброс оценок между прогонами. Один и тот же документ может получить 7/10 и 3/10 в разных запусках. Это несправедливо для студентов.

## Decision

`evaluate_content` запускает LLM 3 раза и берёт медианный результат по итоговому баллу.

## Consequences

**Positive:**
- Значительно выше стабильность: медиана устойчива к выбросам
- Один аномально низкий/высокий прогон не определяет оценку

**Negative:**
- Стоимость evaluate_content ×3 (~0.03-0.09 USD вместо 0.01-0.03)
- Latency ×3 для оценки контента (~15-45s вместо 5-15s)
- При 5 критериях: 15 LLM вызовов вместо 5

**Trade-off:**
- При 200 PR/семестр: +4-12 USD за весь семестр
- Стабильность оценок важнее экономии
- Можно уменьшить до 1 прогона для пересдач (recheck) если latency критична
